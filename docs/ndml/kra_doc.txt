Introduction
This document provides the details of integration of 4 distinct services within our API. Each Service serves a unique purpose and requires specific parameters for the successful execution. There are following services :
KYC registration to register a new KYC record.
KYC modification to modify an existing KYC
KYC inquiry to get the status of the KYC
KYC download to get the documents of the KYC
Each service first requires authentication through encrypted password.
Authentication & KYC Upload/KYC Registration
This is the first service that will register a new KYC. Before uploading the records for KYC registration, you will have to get the encrypted password through “getPassword”

KYC Status Inquiry

There are two methods available for KYC status inquiry.
Method 1: panInquiryDetails  :This method provides only latest status.
Method 2: panInquiryDetailsTwo : This method provides both registration & modification status. We recommend you to integrate this method as it will provide you both status of the PANs.
The request for both the methods have same attributes. In Method two, Response will have additional tag APP_UPDT_STATUS to provide for modification status.

As explained above, you will have to get the encrypted password as defined above in authentication

Errors and Solution

Invalid Requester IP
 	Solution: Provide public IP instead of System IP. (To get public IP, Type in google what is my ip)

Authentication Failed
If User is not authorized to access this method, the following message is returned in ERROR tag. ERROR>AUTHENTICATION FAILED.</ERROR>
Solution: Make sure that your User Id, Password & Passkey is correct.

Unregistered/Inactive mobile number
Similarly, if the mobile no. used to send the request is either not registered or inactive, then the same will be reflected in the ERROR tag.
<ERROR>UNREGISTERED MOBILE NO.</ERROR>
<ERROR>INACTIVE MOBILE NO.</ERROR>
Solution- Make sure that mobile number is registered in system.

        <?php
        print_r("\r\n@@@@@@@@@@      Calling NDML KRA WebService      @@@@@@@@@@\r\n");
        $wsdl = 'http://121.242.223.211/sms-ws/PANServiceImplService/PANServiceImplService.wsdl';
        $options = array(
                                'uri'=>'http://schemas.xmlsoap.org/soap/envelope/',
                                'style'=>SOAP_RPC,
                                'use'=>SOAP_ENCODED,
                                'soap_version'=>SOAP_1_1,
                                'cache_wsdl'=>WSDL_CACHE_NONE,
                                'connection_timeout'=>15,
                                'trace'=>true,
                                'encoding'=>'UTF-8',
                                'exceptions'=>true,
                ); 
        $soap = new SoapClient($wsdl, $options);
//        print_r($soap->__getFunctions());
        try {
            $userId = '<<userID>>';
            $password = '<<password>>';
            $passKey = '<<passkey>>';
            $passcode_params = array('arg0' => $password, 'arg1' => $passKey);
            $encPass = $soap->getPasscode($passcode_params);           
            $encPassword = $encPass->return;
            print_r("@@@@@@@@@@      Encrypted Password  :->      ".$encPassword."\r\n");           
            $xml_request =  '
                            <APP_REQ_ROOT>
                                 <APP_PAN_DOWN>
                                     <APP_PAN_NO>BODPA7810Q</APP_PAN_NO>
                                     <APP_PAN_DOB>01011900</APP_PAN_DOB>
                                     <APP_MOBILE_NO><<mobile number>></APP_MOBILE_NO>
                                     <APP_REQ_NO>1234567890</APP_REQ_NO>
                                 </APP_PAN_DOWN>
                             </APP_REQ_ROOT>
                             ';
            $params = array('arg0' => $xml_request, 'arg1' => $userId, 'arg2' => $encPassword, 'arg3' => $passKey);
//            $data = $soap->panInquiryDetails($params)->return;
            $data = $soap->panDownloadDetails($params)->return;
            print_r("@@@@@@@@@@      XML Response :->      \r\n");
            print_r($data);
        }
        catch(Exception $e) {
            print_r("@@@@@@@@@@      Exception occured :->      ".$e->getMessage()."\r\n");
            die($e->getTraceAsString());
        }
        die;

User creation and Service Registration

Go to website- https://kra.ndml.in/kra-web/ and click on MI Login option and follow below steps

Admin user will login for creation of new user option and select System Security- User Maintenance- New
On the user profile page, select the user type as Functional User and fill in the other mandatory details. (Leave Issuer CA and Certificate No fields as it is in case of password base login).
Group id should be 100 and Login type should be password based.
Change the password of the user by login through website. System will prompt you to change the password while first login.

Please follow below steps for API user registration in admin login

Admin user has functionality in ‘System Security’ as ‘Mobile Registration’ to register users and their mobile number for this facility.
Intermediary administrator has to select Service Type option from drop down as ‘ALL’ services.
Intermediary administrator has to ensure that the mobile number registered for this facility is that of a valid user
